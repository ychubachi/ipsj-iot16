\section{はじめに}
高度専門職には，その専門分野における知識・スキルを最新に維持するための継
続教育が必要であることが指摘されている\cite{drucker03}．産業技術大学院大
学（以下，本学）においても，高度専門職を養成し，継続的に育成するための教
育機会を提供している\cite{ishijima10,chubachi10}．

図\ref{stages}に示すとおり，本学では入学から卒業までの在籍期間を「2nd
Stage」に位置づけている．すなわち，入学前の1st Stage，終了後を3rd Stageと
して捉え，各ステージに対して幅広く教育を行うことをミッションとする．本学
が教育の対象とする領域は「情報システム」及び「ものづくり」のアーキテクト
を育成することであり，高度専門職に対して常に新鮮な知識を提供し続けること
を狙っている．

この目的を達成するためには，キャンパスにおらず，物理的・時間的に距離のあ
る対象者に対しても教育を行うための仕組みが必要である．これには，ICTを活用
した教育用のインフラストラクチャーを整備しなくてはならない．本学では，20

以下，・・・では，本学のシステムのユーザ像を説明する．

これらの取り組みが学生の構成にも反映され，本学産業技術研究科では，2011年
度の学生の約60\%が社会人，平均年齢は 33.2歳であり，著者らが所属する情報アー

特徴を表!!! に示す．

本学が設置する大学院修士課程は図\ref{stages}の第2段階に相当するが，本学で
は，修士課程の学生以外の一般の知的労働者にも継続教育の機会を提供している．
以下に，本学が，学継続教育の対象である社会人に対して整備してきた学修環境
の抜粋を示す。

\begin{itemize}
 \item 単位バンク制度（入学前に科目等履修生制度で履修した分の授業料が入
       学後に返還される）
 \item 各種勉強会（InfoTalk等）開催\cite{infotalk, infotalk2}
 \item 履修証明プログラムの開催（短期間で専門分野の学修）
 \item 附属図書館の地域企業技術者への開放
\end{itemize}


本学の産業技術研究科の収容定員が2専攻2学年で200名であるのに対し，科目等
履修生制度で履修している学生数は平成21年度が68名，平成22年度が72名，平成
23年度51名である。本学の単位バンク制度を利用すれば，入学前に科目等履修生
制度で履修した分の授業料が入学後に返還される。平成22 年度は20名が，平成
23年度は21名がこの制度を利用し，入学前に平均約3.5科目（最高10科目）を履
修してから大学院に入学している。

\section{本学のICT環境の概要}

本学では，電子メールから始まり，無線LAN・グループウェア・遠隔授業，LMS，
授業動画コンテンツ配信，iPBL，サテライトキャンパス等のICT環境を整備して
いる。以下に概要を列挙する。（学生が円滑にPC・ICTを活用できる環境の整備，
時間及び場所の制約を減らし，授業以外は無理に大学に来る必要が無い）

\begin{itemize}
 \item 高速インターネット接続，無線LAN，電源
 \item 電子メール
 \item グループウェア
       \begin{itemize}
	\item 掲示板（連絡事項，就職情報等），電子会議，申請書類
	\item スケジューラ，施設予約
       \end{itemize}
 \item 授業支援（LMS）
       \begin{itemize}
	\item 授業資料（PDF等）配布
	\item 課題提出
       \end{itemize}
 \item 授業動画配信（参考文献）
       \begin{itemize}
	\item すべての授業が収録され，インターネット経由で配信されている。
       \end{itemize}
 \item PBL（Project based Learning）支援（iPBL: infrastructure for PBL）
       \begin{itemize}
	\item プロジェクト管理（協働作業支援） → ファイル，ソースコード，
	      タスク，スケジュール等の管理
	\item プロジェクト活動の評価 → 活動の計画・実績，週報，SA，成果物，成績
       \end{itemize}
 \item 学修成果の蓄積（ポートフォリオ）（参考文献）
       \begin{itemize}
	\item 学習成果（作品，成績，評価）→ 業務実績相当
	\item 評価（教員）
	\item 改善（学生）→ Design-Publish-Collect-Reflectサイクル
	\item 成果の公表（会社あるいはコミュニティ）
       \end{itemize}
 \item ほか
       \begin{itemize}
	\item 学生授業評価，PBL配属希望等はWebから回答・登録が可能（参考
	      文献）
	\item Facebook，Twitter等の活用（コネクション構築支援等）（参考文献）
       \end{itemize}
 \item 遠隔授業
       \begin{itemize}
	\item 一部の授業は秋葉原サテライトキャンパスでも受講できる。
       \end{itemize}
\end{itemize}

\section{おわりに}
本研究では，小規模で，歴史が浅い大学院大学で，学生及び修了生ができるだけ
効率的にまた効果的に，相互にコネクションを構築することができる仕組みを
ICT を活用して実現した．以下に，特長を整理する．
外部アカウント・・・\cite{koyama11}

(1)	Facebook等の普及しているソーシャルメディアを活用しているため，利用者が新規にアカウントを取得する手間や，使用法を新たに修得する必要が無い．

(2)	既存のソーシャルメディアに準備されている機能はできるだけ流用しているので，開発コストを軽減することができた．

(3)	利用者各自のソーシャルグラフの範囲外の相手を，在籍年時，専攻，専門分野等から検索し，コネクションを構築することができる．また，複数のソーシャルアカウントの情報が集約されているので，日常使っているソーシャルメディアが違う相手を発見することができる．

(4)	専門職を対象に，職務経歴をプロジェクト単位で登録することができる．実際の経験してきた業務の内容，規模，技術，成否等をから，専門職の専門分野，レベルを示すことができ，今後のキャリアアップに結び付けることが期待できる．

今後の課題には（1）利用者からの使い勝手・機能等の要望を反映する，（2）利用者の範囲を修了生，創造技術専攻に拡げる，（3）専門職・技術者を雇用する側の企業からの参加を考慮する，（4）知識・スキルを取り扱う機能を準備する，（5）創造技術専攻の専門分野[ ]を対象にした職務経歴の記録を考慮する，等がある．
